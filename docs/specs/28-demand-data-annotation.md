# Spec 28：AI需求量数据标注

**版本**：v1.0
**状态**：📋 待实现
**创建日期**：2026-06-10

---

## 1. 需求描述

### 1.1 背景
AI分析中"市场容量"维度会输出如"月搜索量约50万"的数据，但系统从未查询过真实搜索量。用户可能误以为这是真实数据。

### 1.2 核心需求
1. AI输出的需求量数据必须标注为"AI估算"
2. 如有Google Trends数据，展示真实趋势方向
3. 分析卡片中区分"AI估算"和"真实数据"

---

## 2. 系统设计

### 2.1 改动文件清单

| 文件 | 改动类型 | 说明 |
|------|---------|------|
| `src/analyzer.py` | **修改** | Prompt中要求AI标注"估算"；reason字段加前缀标记 |
| `app.py` | **修改** | 展示时区分AI估算和真实数据，用不同样式 |

### 2.2 Prompt修改

在 BATCH_SYSTEM_PROMPT 中增加：
```
注意：你无法获取真实搜索量数据。请基于评论数、排名、类目热度进行合理推估，
并在reason中使用"估计"、"推测"等词汇，不要给出精确数字（如"50万"）。
应使用范围表述（如"评论数过万，推测月搜索量在10万-50万级别"）。
```

### 2.3 UI标注

```python
reason = dim.get("reason", "")
if any(kw in reason for kw in ["估计", "推测", "约", "大概"]):
    st.caption("🔍 AI估算")
```
