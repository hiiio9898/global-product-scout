## 1. 需求描述

### 1.1 背景
当用户打开应用时，直接面对一个空白的"开始分析"按钮，缺乏全局视角。Dashboard 首页提供数据概览——分析了多少产品、推荐率如何、TOP5 推荐产品有哪些——让用户一眼了解全局，再决定是否深入分析。

### 1.2 核心需求
1. 新增"📊 Dashboard"页面作为导航选项（默认页面或第三选项）
2. 顶部 4 个指标卡片：总分析产品数、今日推荐数、平均市场容量分、平均利润潜力分
3. TOP5 推荐产品卡片列表（标题、价格、评分、verdict）
4. 最近一次抓取的时间信息

### 1.3 设计原则
- 信息密度高，一屏展示核心指标
- 风格简洁商务
- 数据来自数据库（本地）或 session state（Cloud）

## 2. 系统设计

### 2.1 改动文件清单

| 文件 | 改动类型 | 说明 |
|------|---------|------|
| `app.py` | **修改** | 侧边栏导航新增"📊 Dashboard"；新增 `_render_dashboard_page()` 函数 |

### 2.2 数据流

```
app.py 侧边栏导航 → "📊 Dashboard"
        ↓
_render_dashboard_page()
        ↓
读取数据库 / session state 数据
        ↓
计算指标 + 筛选 TOP5
        ↓
st.metric × 4 + st.columns 卡片列表
```

## 4. 验收标准

| # | 验收条件 | 验证方法 |
|---|---------|---------|
| 1 | 侧边栏有"📊 Dashboard"选项 | UI 验证 |
| 2 | 4 个指标卡片显示正确数值 | UI 验证 |
| 3 | TOP5 推荐产品列表显示正确 | UI 验证 |
| 4 | 无数据时显示友好提示 | 删除 DB 后验证 |
| 5 | `python -m py_compile app.py` 通过 | 终端执行 |
| 6 | 不影响现有实时选品和历史记录页面 | UI 验证 |
