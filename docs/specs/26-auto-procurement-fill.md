## 1. 需求描述

### 1.1 背景
当前用户需要：点1688查价 → 读价格 → 关闭弹窗 → 手动输入采购成本 → 看利润。20个产品要重复40次操作，是最耗人工的环节。

### 1.2 核心需求
1. 1688查询成功后，自动将价格中位数填入采购成本字段
2. 自动触发利润重新计算
3. 查询失败时不填入，保持用户手动输入
4. 用户可覆盖自动填入的值（手动输入优先）

---

## 2. 系统设计

### 2.1 改动文件清单

| 文件 | 改动类型 | 说明 |
|------|---------|------|
| `app.py` | **修改** | 1688查询回调中自动写入 `procurement_cost` 并触发利润计算 |
| `app.py` | **修改** | 利润计算区增加"已自动填入"标记，用户可手动覆盖 |

### 2.2 自动回填逻辑

```python
# 在 _render_live_page() 的 1688 查询结果处理中
if result_1688 and result_1688.get("price_range"):
    mid_price = result_1688["price_range"]["mid"]
    if mid_price > 0:
        st.session_state[f"proc_{i}"] = mid_price
        st.toast(f"已自动填入采购成本: ¥{mid_price:.2f}", icon="✅")
```
